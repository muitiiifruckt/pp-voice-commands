# Промпты к ассистенту ИИ (Cursor) при разработке проекта

Документ для отчёта: **какие задачи решались с помощью ИИ** и **что сделано автором вручную**. Формулировки ниже приведены в сжатом виде (как запросы в чат), в реальном Cursor-диалоге часто добавлялись уточнения, фрагменты логов и пути к файлам.

**Инструмент:** Cursor, режим агента с доступом к коду репозитория.

---

## Перечень типовых запросов (промптов)

| № | Запрос (суть промпта) | Зачем / результат |
|---|------------------------|-------------------|
| 1 | Сверь проект с критериями сдачи: исходники, техописание, демо, инструкция развёртывания — что уже есть, чего не хватает. | Структурировать отчёт под баллы. |
| 2 | Добавь автотесты для бэкенда (pytest, FastAPI TestClient) и для фронта там, где уместно. | Каркас `backend/tests/`, примеры запросов к API. |
| 3 | Фронт на Vite показывает только «Загрузка…» — разберись, почему не уходит запрос или не рендерится приложение после логина. | Отладка жизненного цикла `fetchMe` / состояния загрузки. |
| 4 | Пусть результат распознавания после `POST /api/voice/upload` писался в логи сервера, чтобы видеть текст без UI. | Логирование в `asr.py` / обработчик загрузки. |
| 5 | Почему при старте распознавания так долго грузятся логи Vosk (HCLG, RNNLM и т.д.)? | Объяснение разницы `small-ru` vs полная модель. |
| 6 | В интерфейсе пустой текст распознавания, хотя длительность аудио есть — запись «как будто пустая». | Диагностика тихого сигнала, доработка пайплайна ASR и подсказок в UI. |
| 7 | В других программах микрофон пишет нормально, в браузере — тишина; исправь захват. | Выбор устройства `getUserMedia`, индикатор уровня, снятие жёстких `constraints`. |
| 8 | Залей в GitHub содержимое папки `demo` (видео), сделай push. | Коммит, **Git LFS** для MP4 > 100 МБ. |
| 9 | В README укажи, что на демо использовалась модель `vosk-model-ru-0.42`. | Точность описания для проверяющего. |
| 10 | Создай отдельную папку с описанием использованных промптов для отчёта; покажи, что работа не сводилась к одному запросу «сделай всё». | Этот файл. |

Дополнительно в ходе работы использовались **короткие уточняющие** сообщения в том же чате: «сделай», «почему так», «запусти тесты», вставки traceback и скриншотов — без них ассистент не мог бы править нужные файлы.

## Что делалось **самостоятельно** 

- Постановка предметной области и сценариев (какие команды и идентификаторы нужны оператору).
- Установка **Python**, **Node.js**, **FFmpeg**, создание venv, `pip install`, `npm install`, запуск `uvicorn` и `vite`.
- Скачивание и распаковка моделей Vosk, переключение между **small-ru** и **ru-0.42**, настройка `VOSK_MODEL` / `VOSK_MODEL_PATH` в `.env` (в т.ч. обход проблем с путями в Windows).
- Ручная проверка сценариев: логин, запись, история, подтверждение, права админа/оператора.
- Запись **демо-видео**, проверка воспроизведения, загрузка в репозиторий.
- Решение, что включать в отчёт и как формулировать требования методички; правки **своими словами** в `docs/TECHNICAL_SOLUTION_RU.md` и README.
- Регистрация репозитория на GitHub, настройка доступа, при необходимости — разбор ошибок push/LFS.

Итог: ИИ использовался как **ускоритель** для рутинных правок, тестов и отладки по логам, а не как замена понимания задания и проверки работы системы на своей машине.
